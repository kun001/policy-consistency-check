# 政策条款一致性检查工具 - 项目架构与技术栈分析（简化版）

## 项目概述

这是一个基于React的政策条款一致性检查工具，专注于国家政策文件库的管理和检索。该工具提供政策文件库管理功能，通过集成 Weaviate 向量检索与后端 API，实现政策文件检索与查看。项目采用简洁的单页面设计，专注于核心的政策文件管理功能。

### 核心功能模块

#### 1. 政策文件库 (PolicyLibrary)

- **功能**: 展示和管理国家政策文件
- **特性**:
  - 文件搜索和筛选
  - 分类管理
  - 文件详情查看
  - 集成 Weaviate 向量检索与后端 API
  - 数据刷新功能
- **数据源**: Weaviate 向量数据库 / 后端检索 API

#### 2. 政策详情 (PolicyDetail)

- **功能**: 展示单个政策文件的详细信息
- **特性**:
  - 政策内容展示
  - 返回列表功能
  - 详细信息查看

#### 3. API服务层 (weaivateApi.js)

- **功能**: 封装 Weaviate 与后端 API 调用
- **主要接口**:
  - `getNationalPolicyData()` - 获取国家政策数据
  - `getLocalPolicyData()` - 获取地方政策数据
  - `transformNationalPolicyData()` - 数据格式转换（国家）
  - `transformLocalPolicyData()` - 数据格式转换（地方）

## 外部服务集成

### 向量检索与后端 API

- **服务地址**: `http://127.0.0.1:10010/api`（通过前端代理 `/api`）
- **认证方式**: 后端接口鉴权（Token/Session，视实现而定）
- **检索引擎**: Weaviate 本地/私有部署
- **功能**: 文档检索、内容匹配、政策数据查询

### 数据格式

政策文件数据结构包含：

- 基本信息：标题、分类、日期、状态
- 内容结构：章节、条款、关键词
- 统计信息：文档块数、搜索次数
